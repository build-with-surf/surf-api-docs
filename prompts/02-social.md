## 1. KOL 喊单追踪系统

### Prompt

```
帮我写一个 KOL 喊单追踪 Bot，功能要求：

1. 监控以下 KOL 的 Twitter/X 动态：
   {HANDLE_1}
   {HANDLE_2}
   {HANDLE_3}

2. 每 {INTERVAL} 分钟扫描一次
3. 检测喊单时，输出：
   - KOL 名称、发布时间
   - 提及的 Token 名称和代币符号
   - 方向判断（做多/做空/中性）
   - 发布时的代币价格

4. 喊单后追踪：
   - 1h / 6h / 24h 后代币价格变化
   - 链上是否有大额跟单信号（> ${THRESHOLD}）
   - DEX 成交量是否放大（> 基准 {X}%）

5. 输出格式：
   - 控制台实时输出
   - 保存到 {OUTPUT_PATH}（CSV/JSON）
   - 触发 Alert 时发到 {NOTIFICATION}

代码要包含 Twitter API / 第三方数据源接入逻辑，以及防重复处理。
```

### 预期输出
可运行的 Python Bot + Twitter 数据接入代码

### 适用场景
- 跟单策略研究
- KOL 影响力评估
- 社群喊单监控

## 2. 社交媒体舆情监控系统

### Prompt

```
帮我写一个加密市场舆情监控系统：

1. 监控关键词：{KEYWORD_1}, {KEYWORD_2}, {KEYWORD_3}
   （可替换为具体 Token、项目或叙事名称）

2. 数据来源：Twitter/X、Telegram、Discord

3. 分析维度：
   - 讨论量趋势（按小时/天统计）
   - 情绪分类（正面/负面/中性 占比）
   - 影响力加权分数（粉丝数 × 互动率）
   - 核心讨论者识别（谁是意见领袖）

4. Alert 触发条件：
   - 任一关键词讨论量较前日增幅 > {PERCENTAGE}%
   - 负面情绪占比 > {PERCENTAGE}%
   - 出现 {SPECIFIC_EVENT}（如：rug/黑客/崩盘 等关键词）

5. 输出：
   - 实时舆情仪表盘（控制台或 Web）
   - 每日舆情报告（Markdown）
   - 触发 Alert 时推送 {NOTIFICATION}

代码要包含数据采集（Web Scraping / API）、情绪分析和定时任务。
```

### 预期输出
完整的 Python 舆情分析系统

### 变体 Prompt

```
我想监控 "{NARRATIVE}" 叙事的社交热度，当讨论量突然放大时提醒我。
帮我搭一个系统，追踪 Twitter 上所有提及这个叙事的推文，分析整体情绪变化趋势。
```

## 3. 多 KOL 影响力对比分析

### Prompt

```
帮我写一个 KOL 影响力对比分析工具：

1. 对比分析以下 {N} 个 KOL：
   {HANDLE_1}, {HANDLE_2}, {HANDLE_3}

2. 分析维度：
   - 粉丝数量和互动率（点赞/转发/评论）
   - 近 {TIME_RANGE} 的喊单频次
   - 喊单后代币平均涨幅（1h / 24h）
   - 粉丝质量（真粉 vs 水军比例估算）
   - 内容发布频率和活跃度

3. 排名输出：
   - 综合影响力评分（1-100）
   - 各维度单独排名
   - 适合跟单评级（强烈推荐/可参考/谨慎）

4. 输出格式：
   - 表格对比（控制台）
   - 生成 {FORMAT} 报告

代码要可直接运行，包含数据获取和评分算法。
```

### 预期输出
可运行的 Python 分析脚本 + 评分算法

## 4. 舆情-价格相关性分析

### Prompt

```
帮我写一个舆情与价格相关性分析工具：

1. 输入：{TOKEN_SYMBOL}

2. 抓取数据（过去 {TIME_RANGE}）：
   - 该 Token 的社交媒体讨论量（Twitter/Telegram）
   - 该 Token 的价格数据（小时级别）
   - 恐惧贪婪指数

3. 分析：
   - 讨论量与价格的相关性（正相关/负相关/无相关）
   - 讨论量峰值是否领先/滞后于价格峰值（ Granger 因果检验思路）
   - 极端舆情（极度乐观/极度恐慌）与价格顶底的对应关系

4. 输出：
   - 可视化图表（讨论量 vs 价格 叠加图）
   - 相关性系数
   - 关键发现总结（3-5 条）

代码要包含数据获取、统计分析和可视化。
```

### 预期输出
Python 分析脚本 + Matplotlib/Plotly 可视化

## L2 最佳实践

| 原则 | 说明 |
|------|------|
| **指定数据源** | "Twitter API" / "第三方数据平台"，AI 需要知道从哪里取数据 |
| **设置合理阈值** | Alert 阈值太高会漏报，太低会 Spam |
| **Twitter 数据限制** | Twitter API 免费版有频率限制，记得在 Prompt 里说明 |
| **先单 KOL 再批量** | 先跑通一个 KOL 的追踪，再扩展到多 KOL |
| **合规提示** | 喊单追踪仅供研究，跟单需谨慎 |
