## 1. 链上情绪交易信号系统

### Prompt

```
帮我搭建一个链上情绪交易信号系统，功能要求：

## 核心逻辑
当以下条件同时满足时，产生"买入信号"：
- 恐惧贪婪指数 < 25（极度恐慌）
- whale_activity = increasing（鲸鱼开始买入）
- trading_activity 从 low 变为 high（成交量放大）
- {TOKEN} 价格在过去 7d 跌幅 > {PERCENTAGE}%

当以下条件同时满足时，产生"卖出信号"：
- 恐惧贪婪指数 > 75（极度贪婪）
- whale_activity = decreasing（鲸鱼开始卖出）
- 项目方地址向交易所转入 > ${AMOUNT}

## 数据来源
- 情绪数据：Surf Studio API
- 价格数据：{PRICE_API}（CoinGecko/Binance API）
- 链上数据：Surf Studio 链上追踪

## 输出要求
1. 实时信号监控（控制台输出）
2. 信号历史记录（CSV，含时间戳、触发条件、信号类型）
3. 胜率回测模块：
   - 输入历史数据（过去 {BACKTEST_DAYS} 天）
   - 输出：信号总次数、盈利次数、胜率、平均盈利/亏损
4. 信号 Alert 发到 {NOTIFICATION}

## 风险控制
- 设置止损：信号触发后 {STOP_LOSS}% 强制止损
- 设置止盈：信号触发后 {TAKE_PROFIT}% 自动止盈
- 每日最大信号数：{MAX_SIGNALS_PER_DAY}

代码要完整可运行，包含完整的信号逻辑、风险控制和回测模块。
```

### 预期输出
完整的交易信号系统（含信号生成逻辑 + 回测模块）

### ⚠️ 重要声明
```
⚠️ 本系统仅供研究参考，不构成投资建议。
使用前请务必进行充分回测，了解所有风险。
```

## 2. 全自动链上研究 Agent

### Prompt

```
帮我搭建一个全自动链上研究 Agent，功能：

## Agent 架构
设计以下子 Agent，协同工作：

1. 【任务规划 Agent】
   - 接收用户的研究目标
   - 拆解任务为子任务
   - 分配给其他 Agent 执行
   - 汇总结果输出最终报告

2. 【数据采集 Agent】
   - 抓取链上数据（地址活动、TVL、Gas）
   - 抓取社交数据（Twitter/Telegram）
   - 抓取价格数据

3. 【分析 Agent】
   - 情绪分析（Sentiment Analysis）
   - 鲸鱼行为分析
   - 异常检测（异常大额转账、rug 前兆）

4.【报告生成 Agent】
   - 汇总所有分析结果
   - 生成结构化研究报告
   - 给出结论和建议

## 工作流程
用户输入研究目标 → 任务规划 Agent 拆解 → 各 Agent 并行执行 → 报告生成 Agent 汇总

## 输入示例
"帮我研究 {PROJECT/TOKEN} 当前是否值得买入，重点关注链上鲸鱼动向和社区情绪"

## 输出
一份完整的链上研究报告，包含：
- 执行摘要（3 句话）
- 各维度详细分析
- 综合结论和建议

代码要使用异步并发执行（asyncio），各 Agent 之间通过消息队列通信。
```

### 预期输出
多 Agent 协作系统（含任务调度、消息传递、报告生成）

## 3. 跨链监控 & 跨市场套利监控系统

### Prompt

```
帮我搭建一个跨链套利机会监控系统：

## 监控范围
同时监控 {CHAIN_A}、{CHAIN_B}、{CHAIN_C} 这 {N} 条链

## 监控指标
1. 同一 Token 在各链 DEX 的价格差异（价差 %）
2. 跨链桥的资金的流向和流量
3. 各链 Gas 费用对比
4. 各链的 TVL 变化趋势

## 套利信号检测
当以下条件满足时，输出 Alert：
- 同一 Token 在两个 DEX 的价格差 > {ARBITRAGE_THRESHOLD}%
- 减去预估 Gas 成本后利润 > {PROFIT_THRESHOLD}%
- 流动性足够（各池 TVL > ${MIN_LIQUIDITY}）

## 输出
1. 实时价差表格（控制台，每 {INTERVAL} 秒刷新）
2. 套利机会 Alert（含：Token、两个交易所、价差、预估利润）
3. 历史套利机会记录（CSV）
4. Alert 发到 {NOTIFICATION}

代码要包含：
- 多链数据并发获取（asyncio）
- 实时价格对比逻辑
- 套利利润计算模型

⚠️ 实际套利需考虑滑点、Gas 波动、执行延迟等，本系统仅供机会发现参考。
```

### 预期输出
跨链套利监控系统（含多链并发数据抓取）

## 4. DeFi 协议健康度评分系统

### Prompt

```
帮我搭建一个 DeFi 协议健康度评分系统：

## 输入
{PROTOCOL_NAME} 或合约地址 {CONTRACT_ADDRESS}

## 评分维度（每项 0-100，加权平均）

| 维度 | 权重 | 评估内容 |
|------|------|---------|
| 流动性 | 20% | TVL、深度、滑点 |
| 安全性 | 25% | 合约审计、多签权限、暂停机制 |
| 活跃度 | 15% | 日活跃用户、交易笔数、Gas 消耗 |
| 增长性 | 15% | TVL 变化、用户增长趋势 |
| 社区 | 10% | 社交讨论量、KOL 态度 |
| 风险 | 15% | 集中度风险、项目方持仓、rug 可能性 |

## Alert 条件
- 综合评分 < 40：标记为"高风险"
- 任一维度评分 < 20：标记为"异常"
- 评分骤降 > 20 分：标记为"大幅恶化"

## 输出
1. 综合评分（0-100）+ 等级（AA/A/BB/B/C/D）
2. 各维度雷达图
3. 评分变化趋势（过去 {TIME_RANGE}）
4. 风险点详细说明
5. 与同类协议的横向对比

代码要包含评分算法、可视化（雷达图）和定时监控。
```

### 预期输出
DeFi 健康度评分系统（含评分算法 + 可视化）

## 5. 叙事轮动自动追踪流水线

### Prompt

```
帮我搭建一个叙事轮动自动追踪流水线：

## 目标
发现市场热点的切换规律，追踪资金在不同叙事间的流动

## 监控叙事列表
- 叙事 1：{NARRATIVE_1}（相关 Token: {TOKENS_1}）
- 叙事 2：{NARRATIVE_2}（相关 Token: {TOKENS_2}）
- 叙事 3：{NARRATIVE_3}（相关 Token: {TOKENS_3}）
（可自定义添加更多叙事）

## 追踪指标
每个叙事：
1. 相关 Token 的价格变化（7d、24h、1h）
2. 社交讨论量（Twitter/Telegram）
3. TVL 变化（如果是 DeFi 协议）
4. KOL 提及频次

## 轮动判断逻辑
- 当叙事 A 的所有指标连续 {CONSECUTIVE_PERIODS} 超过叙事 B 时，判断为"资金轮动"
- 输出"叙事 A → 叙事 B 的轮动信号"
- 标记轮动阶段：萌芽 → 爆发 → 分化 → 消退

## 输出
1. 叙事热力图（实时可视化）
2. 轮动信号 Alert
3. 每日叙事轮动报告（Markdown）
4. 历史叙事表现排行榜

代码要包含：
- 多叙事并发监控
- 轮动算法
- 热力图可视化

```

### 预期输出
叙事轮动追踪系统（含轮动算法 + 热力图）

## L3 最佳实践

| 原则 | 说明 |
|------|------|
| **系统越复杂，Prompt 越要拆** | 不要试图一个 Prompt 生成整个系统，分模块搭 |
| **先回测再实盘** | 所有交易系统必须先回测验证逻辑正确性 |
| **设置熔断机制** | 连续亏损 N 次自动暂停，防止极端行情 |
| **Credits 意识** | L3 系统数据量大，注意 API 调用成本 |
| **合规第一** | 交易系统必须加免责声明 |
| **分步迭代** | 先跑通核心逻辑，再逐步加风控/回测/通知模块 |
