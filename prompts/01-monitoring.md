## 1. 链上地址监控系统

### Prompt

```
帮我写一个监控 {CHAIN} 上指定地址的 Python 脚本，功能要求：

1. 监控地址列表：{ADDRESS_1}, {ADDRESS_2}, {ADDRESS_3}
2. 每 {INTERVAL} 分钟自动扫描一次
3. 每次扫描检查：
   - 是否有新交易（金额、方向、时间）
   - 是否超过大额阈值 ${THRESHOLD}（标记为重要）
   - 是否转入了交易所地址（标记为可能的抛售信号）
4. 检测到大额转账时，输出：
   - 地址、金额、方向、目标地址、时间戳
   - 简洁的链上行为解读（如："检测到大额转入，可能是吸筹"）
5. 将结果保存到 {OUTPUT_FORMAT}（CSV/JSON/数据库）

代码要可以直接运行，带错误处理和重试逻辑。
```

### 预期输出
可运行的 Python 脚本 + 说明文档

### 适用场景
- 监控项目方钱包，看是否在出货
- 监控鲸鱼地址，看是否在买入
- 监控做市商地址，看资金流向

### 变体 Prompt

```
我想监控 {WALLET} 的链上活动，当它向交易所转入超过 ${AMOUNT} 时提醒我。
帮我写一个监控脚本，设置 {TIME_INTERVAL} 的扫描间隔，结果发到 {NOTIFICATION_CHANNEL}。
```

## 2. 实时数据 Dashboard

### Prompt

```
帮我搭建一个加密市场数据 Dashboard，前端用 {FRAMEWORK}，后端调用 Surf Studio 数据。

功能要求：

1. 展示以下指标：
   - BTC/ETH/SOL 当前价格和 24h 涨跌
   - 恐惧贪婪指数（数值 + 可视化）
   - 市场情绪标签（bullish/neutral/bearish）
   - {CUSTOM_TOKEN} 自定义代币的价格
   - 大户净流量（24h）

2. 数据每 {REFRESH_INTERVAL} 秒自动刷新
3. 支持 {N} 个代币的自定义切换
4. 图表展示：价格走势图 + 成交量柱状图

5. 部署方式：{DEPLOY_PLATFORM}

代码要完整可运行，包含前端页面和后端数据抓取逻辑。
```

### 预期输出
完整的前后端代码 + 部署说明

### 适用场景
- 个人盯盘 Dashboard
- 给团队看的市场看板
- 多指标聚合监控面板

### 变体 Prompt

```
帮我写一个 Dashboard，实时显示 {TOKEN} 的链上情绪指标，包括 fear_greed_index、whale_activity、trading_activity 三个维度，用折线图展示过去 {TIME_RANGE} 的变化趋势。
```

## 3. 自动化早报生成器

### Prompt

```
帮我写一个每日加密市场早报生成器，功能：

1. 每天 {TIME} 自动运行
2. 抓取以下数据：
   - BTC/ETH/SOL + {CUSTOM_TOKENS} 的价格和涨跌
   - 恐惧贪婪指数
   - 过去 24h 市场情绪（bullish/neutral/bearish）
   - 讨论热度最高的 3 个叙事/项目
   - 值得关注的链上异动
3. 生成结构化早报：
   - 今日市场概览（200字以内）
   - 热点叙事（3个）
   - 链上异动（2-3条）
   - 风险提示（如有）
   - 今日关注（数据发布/事件日历）
4. 输出格式：Markdown，保存到 {OUTPUT_PATH}
5. 每天自动发到 {NOTIFICATION}（Telegram/Discord/邮件）

代码要包含定时任务逻辑（用 APScheduler 或 cron）。
```

### 预期输出
完整的 Python 脚本 + 定时任务配置 + 通知集成代码

### 适用场景
- 个人早报订阅
- 社群早报自动推送
- 团队市场简报

## 4. Token 持币分析工具

### Prompt

```
帮我写一个 Token 持币分布分析工具：

1. 输入：{TOKEN_ADDRESS} 或代币符号 {TOKEN_SYMBOL}
2. 分析内容：
   - 持币地址总数
   - 前 10 地址的持币量和占比
   - 巨鲸（持有 > 1%）地址数量
   - 过去 {TIME_RANGE} 新增/减少的持币地址数
   - 持币集中度评分（0-100）
3. 判断逻辑：
   - 如果前 10 地址占比 > 50%，标记为"高度集中"
   - 如果项目方地址持有 > 30%，标记为"项目方控盘风险"
4. 输出：
   - 控制台表格展示分析结果
   - 生成 {OUTPUT_FORMAT} 报告

代码要可以直接运行，包含 API 调用和结果解析。
```

### 预期输出
可运行的 Python 分析脚本

## L1 最佳实践

| 原则 | 说明 |
|------|------|
| **一次只搭一个系统** | 不要在一个 Prompt 里同时要 Dashboard + Alert + 报告，分开搭更稳定 |
| **指定技术栈** | "用 Next.js" / "用 FastAPI"，AI 不会猜 |
| **指定输出格式** | "输出 CSV" / "保存到数据库"，AI 更明确 |
| **指定触发条件** | "每小时" / "每天 9 点"，不要模糊 |
| **先跑通再迭代** | 先让 AI 生成最小可用版本，再逐步加功能 |
